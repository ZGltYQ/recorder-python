## Gap Analysis from VERIFICATION.md

### Gap 1: PriorityQueue Cannot Function (CRITICAL)
**Root Cause:** `QueuedQuestion` dataclass (lines 17-36) lacks comparison operators (`__lt__`, `__le__`, `__gt__`, `__ge__`). `asyncio.PriorityQueue` uses `heapq.heappush`/`heappop` which require items to be comparable.

**Impact:** Queue crashes at runtime on second `put_nowait()` call with `TypeError: '<' not supported between instances of 'QueuedQuestion'`.

### Gap 2: Aging Starvation Prevention Not Implemented
**Root Cause:** `_apply_aging()` method (lines 189-194) is an empty stub with only a `pass` statement.

**Impact:** Normal queue items never get priority boost, so starvation prevention per PRIO-03 is not functional.

## Design Decision

The asyncio.PriorityQueue requires comparable items. The cleanest fix is:
1. Add `__lt__` operator to QueuedQuestion that compares by `(priority, timestamp)` tuple - lower priority number (1) comes before higher (2), and within same priority, earlier timestamp comes first.
2. For aging: Since asyncio.PriorityQueue is a heapq-based structure that doesn't support direct iteration/reordering, we need a different approach. Option: track aging by re-queuing items with updated priority, or use a separate tracking mechanism.

Given the constraint that we can't easily iterate the heap, the recommended approach is:
- On each `_apply_aging()` call, for normal-priority items that have waited long enough, temporarily boost their effective priority by re-queueing with a lower effective priority number
- Use a wrapper or track items separately for aging purposes

```python
def __lt__(self, other: "QueuedQuestion") -> bool:
    """Compare by priority first, then timestamp for FIFO within same priority."""
    if self.priority != other.priority:
        return self.priority < other.priority
    return self.timestamp < other.timestamp

def __le__(self, other: "QueuedQuestion") -> bool:
    return self == other or self < other

def __gt__(self, other: "QueuedQuestion") -> bool:
    if self.priority != other.priority:
        return self.priority > other.priority
    return self.timestamp > other.timestamp

def __ge__(self, other: "QueuedQuestion") -> bool:
    return self == other or self > other
```

Implement actual aging logic in `_apply_aging()` method (currently lines 189-194).

The challenge: asyncio.PriorityQueue is a heapq-based structure that doesn't support direct iteration or re-ordering of items in place.

The solution: Since we can't iterate the heap directly, we implement aging by using the `effective_priority` field at dequeue time. The `_apply_aging()` method will trigger a re-prioritization pass by:
1. Cancelling the current worker task
2. Draining the queue and re-queueing items with updated effective priorities
3. Restarting the worker

However, a simpler approach that works with the existing structure: Instead of trying to modify items in the heap (which is complex), we track items' wait time and boost their effective priority at dequeue time. Since `QueuedQuestion.__lt__` compares by `(priority, timestamp)`, we need to modify how we handle aging.

**Alternative (simpler) approach:** Use a separate "aging candidate" list that tracks normal-priority items. When aging timer fires, for items that have exceeded `aging_interval`, temporarily re-assign them a better priority number for ordering purposes.

Actually, the cleanest approach for this codebase:

```python
def _apply_aging(self) -> None:
    """Apply aging to normal queue items to prevent starvation.
    
    Since asyncio.PriorityQueue is a heap, we implement aging by:
    1. Re-queuing normal items that have waited long enough with adjusted timestamp
    2. This effectively moves them ahead in the queue within their priority level
    
    Items get aging boosts based on wait time:
    - Every aging_interval seconds, normal items get timestamp adjustment
    - This makes them appear "earlier" in the FIFO ordering for same-priority items
    """
    if not self._running or self._queue.empty():
        return
    
    # Count how many normal-priority items would benefit from aging
    aged_count = 0
    current_time = time.monotonic()
    
    # We can't iterate the heap directly, so we temporarily transfer items
    # to apply aging, then put them back
    temp_items = []
    while not self._queue.empty():
        try:
            item = self._queue.get_nowait()
            temp_items.append(item)
        except asyncio.QueueEmpty:
            break
    
    # Apply aging: for normal priority items, adjust their effective ordering
    for item in temp_items:
        if not item.is_priority:  # Normal priority item
            wait_time = current_time - item.timestamp
            age_increments = int(wait_time / self.aging_interval)
            if age_increments > 0:
                # Boost by moving timestamp backward (simulates priority improvement)
                # Cap at max_age increments to prevent infinite promotion
                boost = min(age_increments, self.max_age) * self.aging_factor
                # Create new timestamp that makes this item appear earlier
                item.timestamp = current_time - (boost * self.aging_interval)
                aged_count += 1
        
        # Re-queue the item
        self._queue.put_nowait(item)
    
    if aged_count > 0:
        logger.debug("Applied aging to items", aged_count=aged_count)
```

Note: The above approach has an issue - once an item is in the heap with a given priority, changing its timestamp doesn't help because the heap doesn't reorder. A better approach is to use effective_priority in comparison, but that requires more restructuring.

```python
    1. Periodically scanning (via temporary drain) for aged normal items
    2. Moving them to a separate _aged_queue asyncio.PriorityQueue
    3. Worker checks _aged_queue first before main queue

    if not self._running:
        return
    
    # Drain items, check aging, requeue
    temp_items = []
    current_time = time.monotonic()
    
    while not self._queue.empty():
        try:
            item = self._queue.get_nowait()
        except asyncio.QueueEmpty:
            break

            if age_increments > 0 and age_increments >= self.max_age:
                # Item has aged enough - move to front of normal queue
                aged_normal_items.append(item)

    if aged_normal_items:
```

```python
def _apply_aging(self) -> None:
    """Apply aging to normal queue items to prevent starvation.
    
    Since asyncio.PriorityQueue uses heapq which doesn't support in-place reordering,
    aging is implemented via a promotion mechanism:
    1. Every aging_interval, we check for aged normal items
    2. Items that have waited max_age intervals get promoted to priority queue
    3. This is done by temporarily draining and re-queuing
    """
    if not self._running or self._queue.empty():
        return
    
    current_time = time.monotonic()
    promoted = []
    remaining = []
    
    # Temporary drain
    while not self._queue.empty():
        try:
            item = self._queue.get_nowait()
        except asyncio.QueueEmpty:
            break
        
        if not item.is_priority:
            wait_time = current_time - item.timestamp
            age_increments = int(wait_time / self.aging_interval)
            
            # After max_age intervals, promote this item
            if age_increments >= self.max_age:
                # Promote: mark as priority temporarily
                promoted_item = QueuedQuestion(
                    priority=self.PRIORITY_BASE,  # Use priority base (1)
                    timestamp=item.timestamp,  # Keep original timestamp for FIFO
                    message_id=item.message_id,
                    question=item.question,
                    is_priority=True  # Now treated as priority
                )
                promoted.append(promoted_item)
                logger.debug("Item promoted to priority", message_id=item.message_id, age_increments=age_increments)
            else:
                remaining.append(item)
        else:
            remaining.append(item)
    
    # Re-queue all items: promoted first (they're now priority), then remaining
    for item in promoted:
        self._queue.put_nowait(item)
    for item in remaining:
        self._queue.put_nowait(item)
    
    if promoted:
        self._priority_count += len(promoted)
        self._normal_count -= len(promoted)
        self._emit_queue_depth()
```
